YOLA es un agente de IA que opera 100% en tu máquina. No es un chatbot cloud ni un plugin de editor — es un motor autónomo con acceso directo al filesystem, tools de sistema y un loop de decisión propio. Corre sobre Bun, usa TypeScript en toda la pila, y se conecta a cualquier modelo LLM via AI SDK.

100% local.
Cero datos a la nube.

YOLA opera completamente dentro de tu máquina. El motor de agente, la memoria RAG, el index de código, las sesiones — todo corre en tu hardware. No hay servidores remotos procesando tu información. No hay telemetría. No hay vendor lock-in. Es arquitectura local por diseño, no por marketing.

Arquitectura por capas.

YOLA está organizado en 5 capas que van del storage al UI. Cada capa tiene responsabilidades claras, interfaces definidas y un propósito específico. Así es por dentro.

💾
Capa 0
Filesystem — Verdad Absoluta
El disco no miente. YOLA verifica cada operación contra el filesystem antes de actuar. No asume — lee, escribe, confirma. Es la fuente de verdad de todo el sistema. Cada archivo, cada cambio, cada resultado se valida contra lo que realmente está en disco.
"El código en disco no puede ser falseado."
⚙️
Capa 1
Motor de Agente
El loop de decisión que gobierna a YOLA. Recibe contexto del LLM, elige qué tool ejecutar (code_rag, edit, bash, grep, etc.), verifica el resultado, y decide el siguiente paso. Incluye protocolo de evidencia (V/L/?) y memoria automática (RAG inyectada en cada turno).
"Decide. Ejecuta. Verifica. Repite."
🧠
Capa 2
LLM Provider
Capa de abstracción sobre modelos de lenguaje usando Vercel AI SDK (@ai-sdk/*). Soporta 20+ providers: OpenAI, Anthropic, Google Gemini, AWS Bedrock, Azure, y modelos locales via Ollama. Provee streaming, tool calling nativo, y manejo de contexto hasta 1M tokens.
"Un solo agente, cualquier modelo."
🖥️
Capa 3
Desktop Shell — YolaDesktop
Interfaz de escritorio construida con Tauri v2 (Rust + web frontend). Incluye Taskbar, ventanas nativas con drag, notificaciones del sistema, y panel de propuestas. El frontend usa Solid.js + Tailwind CSS con tema oscuro y glassmorphism.
"Nativo, no una web disfrazada."
🌐
Capa 4
Server API & MCP
Servidor HTTP en :4096 que maneja sesiones de agente, plugins MCP (Model Context Protocol), y un bridge opcional de Telegram en :4098. Permite extender YOLA con herramientas externas y comunicación remota controlada vía API REST.
"Extensible por diseño, segura por arquitectura."
🔧

Secuencia de arranque

Cada vez que iniciás YOLA, las 5 capas se activan en orden. Cada una depende de la anterior. Así es el boot sequence real del sistema.

1

💾 Capa 0: Filesystem

Verifica el workspace, monta los directorios del proyecto, y confirma que el índice code-rag existe y está actualizado. El índice contiene 1600+ archivos y 10500+ chunks listos para búsqueda semántica. Si falta, se reconstruye automáticamente.

code-rag · Indexación semántica local
2

🧠 Capa 2: LLM Provider

Se conecta al modelo de lenguaje configurado via Vercel AI SDK. Soporta 20+ providers: OpenAI, Anthropic, Google Gemini, AWS Bedrock, Azure, Ollama (local). El modelo elegido define la capacidad de razonamiento y el tamaño de contexto disponible.

AI SDK · Provider agnóstico
3

⚙️ Capa 1: Motor de Agente

Arranca el loop de decisión: recibe el contexto del LLM, elige la siguiente tool a ejecutar, verifica el resultado contra el filesystem, y decide el próximo paso. Es el cerebro operativo del sistema.

xterm.js + node-pty · Terminal nativa
4

🖥️ Capa 3: Desktop Shell

Se levanta la interfaz de escritorio con Tauri v2. Aparecen las ventanas nativas, la Taskbar, y el panel de propuestas. El frontend Solid.js se monta sobre el webview de Tauri con tema oscuro y glassmorphism.

Tauri v2 · Solid.js · Tailwind CSS
5

🌐 Capa 4: Server API

Inicia el servidor HTTP en :4096 que expone la API REST de YOLA. Maneja sesiones de agente, plugins MCP, y el bridge opcional de Telegram en :4098. También arranca la memoria persistente vía SQLite (drizzle-orm).

HTTP :4096 · MCP · SQLite
6

✅ Sistema listo

Todas las capas están operativas. El motor de agente tiene contexto fresco del code-rag, el LLM provider está conectado, la desktop shell muestra la interfaz, y la API sirve requests. YOLA está funcionando.

YOLA vs. alternativas.

No competimos con asistentes cloud ni plugins de editor. Somos una categoría diferente: agente local autónomo.

✕ Cursor / ChatGPT / Copilot

  • ☁ Requieren conexión a servidores remotos
  • 👁 No ven tu sistema completo — solo el editor o el chat
  • ⏱ Existen solo cuando los abrís (sin daemon persistente)
  • 🔁 Sin memoria entre sesiones (o memoria limitada a chat)
  • 🧩 Propietarios · Código cerrado · Vendor lock-in
  • 💰 Suscripción mensual ($10–20/mes)
  • 🔒 Sin acceso a tools de sistema (edit, bash, filesystem)
  • 🏢 Controlados por corporaciones (OpenAI, Microsoft, Anthropic)

★ YOLA

  • 🏠 100% local — nada sale de tu máquina
  • 👁 Acceso directo al filesystem — lee y modifica archivos reales
  • ⚙️ Motor autónomo con loop de decisión y tools de sistema
  • 📈 Memoria persistente via RAG + SQLite — aprende de cada sesión
  • 🧬 Multi-modelo (20+ providers) — sin vendor lock-in
  • 🌍 Open source (MIT) — código en GitHub
  • 🆓 Siempre gratis — sin suscripción, sin tiers
  • 🔓 Tu control — corre en tu hardware, nadie puede quitártelo
