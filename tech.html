Stack técnico por componente

Cada servicio usa el lenguaje y las herramientas óptimas para su función. Sin sobreingeniería.

YOLA Engine

Núcleo del ecosistema

Motor autónomo de agente IA. Loop de decisión, tools de sistema, conexión a LLMs. Daemon persistente con API REST y WebSocket.

Go SQLite WebSocket Vanilla JS

YolaAvisa

App Android

Cliente móvil para captura de notificaciones. Base de datos local con Room, sincronización vía Retrofit, parsers extensibles.

Kotlin Compose Room Retrofit

YolaPanel

Dashboard web

Panel de administración y monitoreo. Visualización de métricas, logs y estado del ecosistema. Static generated, zero runtime deps.

Next.js TypeScript Chart.js Tailwind

YolaVigila

Agente de monitoreo

Proceso ligero que vigila recursos del sistema, procesos y eventos. Envía alertas vía WebSocket al Engine y notifica a YolaAvisa.

Node.js Express WebSocket systeminformation

Principios de diseño

No son features. Son compromisos arquitectónicos que definen cada línea de código.

01

Soberanía

Tu máquina, tus datos, tus decisiones. YOLA no reporta a ningún servidor externo. No hay telemetría, no hay phone home, no hay puerta trasera.

02

Mínimo

Cada componente hace una cosa y la hace bien. Sin frameworks opinados, sin capas de abstracción innecesarias. El código es el contrato.

03

Local-first

Todo funciona sin internet. El engine, la base de datos, el monitoreo, las notificaciones. La nube es opcional, nunca un requisito.

04

Modular

Puedes usar YOLA Engine solo, o sumar Vigila, Avisa y Panel. Cada componente es independiente, con API claras y contratos definidos.

05

Auditable

Código abierto, sin binarios cerrados. Cada decisión del agente se loggea. Cada transacción se registra. No hay opacidad.

06

Eficiente

17 MB para el Engine. ~8 MB para Vigila. APK de 4 MB para Avisa. Sin Electron, sin runtime pesado, sin contenedores.
